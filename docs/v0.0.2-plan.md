# voice-inbox-daemon v0.0.2 Plan

Date: 2026-03-20
Status: reviewed, ready for implementation
Reviewed by: /plan-eng-review (Claude), pending Codex second opinion

## Goals

1. Accept pre-transcribed text from Android (use OpenAI gpt-4o-mini-transcribe, skip local Whisper)
2. Simplify Journal entry format (remove verbose YAML metadata that wastes AI context)

## Architecture Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| 1 | Transcript storage | DB column `transcript_text TEXT` (schema v3) | Text is a few KB max; no file I/O needed |
| 2 | Journal dedup | HTML comment `<!-- capture:source:id -->` | Invisible to Obsidian/AI, searchable by daemon |
| 3 | Discord/Android format | Both use new simplified format | DRY: no format bifurcation |
| 4 | Metadata line | `_HH:MM via Device · Location_` | Time + device + optional location |
| 5 | Raw audio retention | Keep + 14-day cleanup (unchanged) | Existing policy handles it |

## Data Flow

```
Android app:
  record → OpenAI gpt-4o-mini-transcribe → transcript
  upload: POST /v0/captures (audio + transcript + capture_id + device_id + captured_at)

daemon ingest (server.go):
  accept multipart → read optional "transcript" field
  persist raw audio file → insert captures row with transcript_text

daemon processor (runner.go):
  if transcript_text is present:
    skip ffmpeg normalization
    skip Whisper transcription
    use transcript_text directly
  else:
    existing path: ffmpeg → Whisper → transcript

Journal append:
  new format (both Discord and HTTP):
    ## ログ - HH:MM
    ### 🎤 Voice Inbox

    {transcript text}

    _HH:MM via {device}_
    <!-- capture:{source}:{capture_id} -->
```

## Changes by File

- Read `transcript` form field from multipart (optional)
- Pass transcript_text to CreateCapture
- No change to response shape

### 2. internal/state/store.go
- Add `transcript_text TEXT` column to captures table
- Schema migration v2 → v3: `ALTER TABLE captures ADD COLUMN transcript_text TEXT`
- Add transcript_text to CaptureRecord struct
- Update CreateCapture to accept transcript_text
- Add GetTranscriptText helper or include in GetCapture

### 3. internal/pipeline/runner.go
- In processStoredCapture: check if capture has transcript_text
  - If yes: skip ffmpeg + Whisper, use transcript_text directly, set model to "gpt-4o-mini-transcribe"
  - If no: existing path (ffmpeg → Whisper)
- Update journalContainsCapture to search for `<!-- capture:` HTML comment instead of `capture_key:` YAML

### 4. internal/journal/journal.go
- Rewrite BuildEntry to produce new simplified format:
  ```
  \n## ログ - HH:MM\n### 🎤 Voice Inbox\n\n{transcript}\n\n_{HH:MM via Device}_\n<!-- capture:{source}:{id} -->\n
  ```
- Remove all YAML block generation
- Update EntryInput: add optional Location field for future GPS support
- Keep CaptureKey function (used for HTML comment marker)

### 5. Tests
- internal/ingest/server_test.go: test transcript field acceptance
- internal/journal/journal_test.go: update for new format, test HTML comment presence
- internal/pipeline/runner_integration_test.go: test Whisper skip path, test dedup with HTML comment

## NOT in scope

- GPS/location data from Android (future, format supports it)
- Android app changes (already sends transcript field)
- launchd serve 常駐化 (separate task)
- Transcript editing UI
- Multi-language transcription model selection

## Breaking Changes

- Journal entry format changes from YAML block to simplified format
- Existing entries retain old format; new entries use new format
- Dedup marker changes from `capture_key: "..."` to `<!-- capture:... -->`
  - Old entries can still be found by old marker (backward compat in search)
  - New entries only use HTML comment marker

## Test Plan

```
1. Ingest with transcript field
   → transcript_text stored in DB
   → response unchanged (201 Created)

2. Ingest without transcript field
   → transcript_text is NULL
   → existing Whisper path runs

3. processStoredCapture with transcript_text
   → Whisper skipped
   → Journal entry uses transcript_text
   → model shows "gpt-4o-mini-transcribe"

4. processStoredCapture without transcript_text
   → existing ffmpeg + Whisper path
   → Journal entry uses Whisper output

5. Journal dedup with HTML comment
   → same capture_id not appended twice
   → different source + same ID → both appended

6. Discord entries use new format
   → no YAML block
   → HTML comment dedup works

7. Schema migration v2 → v3
   → existing captures table gains transcript_text column
   → existing rows have NULL transcript_text
```
